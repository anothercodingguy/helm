# Deployment Guide (Render.com)

## Step 1: Push to GitHub
Ensure all files are committed and pushed to your GitHub repository.

```bash
git add .
git commit -m "Prepare for deployment"
git push origin main
```

## Step 2: Create Blueprint on Render
1. Go to [dashboard.render.com](https://dashboard.render.com).
2. Click **New +** -> **Blueprint**.
3. Connect your GitHub repository.
4. Render will automatically detect `render.yaml`.
5. Click **Apply**.

## Step 3: Configure Environment Variables
Render will prompt you for "sync: false" variables. Enter the following manually in the dashboard:

**Backend Service (`saas-backend`):**
- `PHONEPE_MERCHANT_ID`: Your Merchant ID (or sandbox ID)
- `PHONEPE_SALT_KEY`: Your Salt Key (or sandbox key)

**Moltbot Service (`moltbot`):**
- `ANTHROPIC_API_KEY`: Your real LLM API Key (if you integrate one later)

## Step 4: Verify Deployment
1. Wait for all services to build and deploy (Frontend, Backend, Moltbot, DB).
2. Visit the **Frontend URL** (e.g., `https://saas-frontend-xyz.onrender.com`).
3. Signup/Login and test the flow!

## Notes
- **Internal Networking**: The Backend talks to Moltbot at `http://moltbot:7000`. This is private and fast.
- **Database**: Render creates and links the PostgreSQL DB automatically.
- **Secrets**: `JWT_SECRET` is auto-generated by Render.
